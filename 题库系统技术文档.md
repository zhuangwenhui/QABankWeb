# 题库系统 技术文档

> 面向大学院入学考试备考的理工科题库管理系统。本文档记录系统的整体架构、页面结构、数据模型、接口设计、核心功能与改进方向，供接手维护与后续改进使用。

---

## 1. 系统概述

本系统是一个理工科题库管理平台，服务于日本大学院入学考试备考。核心能力是以 LaTeX 存储和渲染数学公式，并基于题库和错题本生成 PDF 试卷。系统当前收录约 938 道题，覆盖 7 门课程。

系统采用服务端渲染的多页应用（MPA）架构：每个功能路由对应一个独立的整页，页面由后端模板渲染，前端交互逻辑以内联脚本嵌入页面。表单与接口请求带 CSRF token 保护，使用同源会话（Session）认证。后端基于 Python Flask + Jinja2 模板，具备 LaTeX 到 PDF 的编译能力。

## 2. 技术栈

界面基于 Bootstrap 5.1.3 与 Font Awesome 6，二者通过 CDN 引入。数学公式渲染使用 MathJax 3（启用 boldsymbol 扩展）。LaTeX 编辑器使用 CodeMirror 5.65.2，配合 stex 语法模式，支持实时预览。

前端自有脚本集中在三个文件：`/static/js/app.js`（应用入口）、`utils.js`（通用工具函数）、`toast.js`（消息提示组件）。页面级交互逻辑内联在各模板中。

PDF 生成依赖 LaTeX 引擎，导出模板以 `.tex` 文件形式存放于服务端。

## 3. 路由与页面

系统包含四个主路由：

- `/questions` — 题目管理。同时是站点默认首页，根路径 `/` 重定向至此。
- `/error_book` — 错题本。
- `/feedback` — 意见反馈。
- `/logout` — 登出。

全站共用固定顶部导航栏（`navbar fixed-top`），包含品牌标识、四个菜单入口，以及右上角的用户下拉菜单（含退出登录）。

## 4. 数据模型

### 题目（Question）

系统的核心实体，字段如下：

| 字段 | 说明 |
|------|------|
| id | 自增主键，页面以 #编号 显示 |
| subject | 课程名称 |
| chapter | 章节名称（承载知识点章节或考试年份） |
| difficulty | 难度，枚举：简单 / 中等 / 困难 |
| source | 题目来源，如"东京工业大学 计算数理" |
| tags | 标签，JSON 数组，如 `["线性代数","固有值"]` |
| question_latex | 题目内容，LaTeX 文本 |
| question_image | 题目图片，支持 image/* 与 PDF |
| solution_latex | 解答内容，LaTeX 文本 |
| solution_image | 解答图片，支持 image/* 与 PDF |
| created_at | 创建时间 |

课程为固定分类：向量解析、备注、复变函数、微分方程、微积分、概率统计、线性代数。

章节字段同时承载知识点章节（如"1 変数関数の微分法"）与考试年份（如 2008–2021）。

### 错题本（ErrorBook）

学生与题目的关联实体，记录某学生标记的错题集合。支持按科目、章节、难度、来源筛选，支持批量移除。每条错题可附加备注（notes）。页面提供总错题数与按科目分布的统计。

### 反馈（Feedback）

反馈工单实体，含状态字段（待处理 / 已处理），以列表形式管理。

## 5. 接口设计

所有接口位于 `/api/` 下，带 CSRF 保护，使用同源会话认证。

| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/questions` | GET | 题目查询与筛选分页 |
| `/api/log_view_question` | POST | 记录题目查看行为，支撑学习统计 |
| `/api/source_exists` | GET | 来源判重校验 |
| `/api/upload_question_image` | POST | 上传题目/解答图片 |
| `/api/delete_question_image` | POST | 删除图片 |
| `/api/error_book/add` | POST | 加入单题至错题本 |
| `/api/error_book/add_batch` | POST | 批量加入错题本 |
| `/api/error_book/remove` | POST | 从错题本移除 |
| `/api/error_book/check_batch` | POST | 批量查询题目是否已在错题本中，用于渲染书签状态 |

题目查询接口的参数集（来自高级搜索面板）：subject、chapter、difficulty、source、search（关键词）、questionId、tagFilter（逗号分隔标签）、dateFrom、dateTo，以及分页参数（每页 10 / 20 / 50 / 100）。

## 6. 功能模块

### 题目管理

**视图切换**：提供表格视图与卡片视图两种展示，通过顶部切换按钮切换。

**筛选与搜索**：基础筛选包含课程、章节、难度、来源、关键词；高级搜索额外提供题目 ID、标签筛选、创建时间范围。

**批量操作**：支持全选、批量删除、批量编辑标签、批量修改来源。选中项通过工具栏集中操作。

**搜索增强**：保存并回放搜索历史；将当前筛选条件保存为预设，供快速加载与管理。

**题目详情**：点击题目公式区打开详情弹窗，分为题目信息、题目内容、题目图片、解答内容、解答图片五个区块，并可展开查看答案。题目卡片提供书签操作以加入或移出错题本，并支持右键上下文菜单。

**题目编辑**：编辑弹窗覆盖全部字段——课程、章节、难度、来源、标签（JSON）、题目 LaTeX、解答 LaTeX、题目图片、解答图片，公式区提供实时预览。

### 错题本与 PDF 生成

错题列表支持与题库一致的筛选维度，支持批量移除，每题可编辑备注、快速预览。页面展示总错题数与科目分布。

PDF 生成为本模块核心。导出配置包含试卷标题、副标题、考试日期、科目、考试时间、总分、注意事项，并可选择试卷模板（custom_exam_template、试卷模板、error_book_template 三套 `.tex` 模板）。确认后由后端以选定模板编译生成试卷 PDF。

### 意见反馈

反馈工单列表，可按状态（全部 / 待处理 / 已处理）筛选，并支持重置筛选条件。

## 7. 前端样式约定

除 Bootstrap 基础类外，系统自定义了以下样式类：

- 布局：`main-content`、`content-area`、面包屑 `breadcrumb-container` / `breadcrumb-custom` / `breadcrumb-separator`
- 题目展示：`question-card` / `question-card-view` / `question-card-item`（卡片视图）、`question-table`（表格视图）
- 公式渲染：`latex-code` / `latex-content` / `latex-preview`
- 难度标签：`difficulty-badge`（中等为黄色、困难为红色、简单为绿色）
- 交互：`batch-toolbar`、`view-toggle` / `view-toggle-btn`、`selected`（选中态）
- 系统消息：`flash-messages`

## 8. 角色与权限

系统区分学生与管理角色。导航中的"总览"入口对学生角色重定向至题目管理页，表明总览为管理侧视图，题库与错题本为学生侧主要工作区。权限体系的具体划分以后端实现为准。

## 9. 改进方向

以下按优先级列出主要改进切入点。

### 9.1 前端代码结构重构（高优先）

当前页面交互逻辑以约 173KB 的巨型内联脚本嵌入模板，函数散落在全局作用域（约 40 余个全局函数），难以维护和测试。建议将内联脚本抽离为独立模块文件，按功能域拆分（题目管理、错题本、搜索预设、公式编辑、批量操作），并引入构建工具统一打包与压缩。此项是后续所有改进的基础。

### 9.2 接口契约与前后端解耦（高优先）

现有页面以服务端渲染为主、接口为辅，接口返回结构未文档化。建议先补齐每个接口的请求/响应契约文档，统一响应格式与错误码，再逐步将高交互页面（题目管理、错题本）改造为接口驱动，为将来前端框架化留出空间。

### 9.3 数据模型规范化（中优先）

章节字段当前混用知识点章节与考试年份两类语义，标签以 JSON 字符串存储。建议将年份拆为独立字段（便于按年份检索与统计），标签改为规范化关联表（支持标签管理、去重、统计），来源建立独立字典表（配合已有的判重接口）。这将显著提升筛选与统计能力。

### 9.4 PDF 生成流程健壮化（中优先）

PDF 依赖服务端 LaTeX 实时编译，存在编译耗时、失败无反馈、并发阻塞的风险。建议引入异步任务队列处理编译，向用户返回生成进度与失败原因，对模板做校验与沙箱化，并缓存相同参数的生成结果。

### 9.5 搜索与性能（中优先）

题库已达千级规模，关键词搜索若为数据库模糊匹配，规模增长后性能会下降。建议为常用筛选字段建立索引，并评估引入全文检索。搜索历史与筛选预设当前存于浏览器本地，建议迁移至后端账户维度，实现跨设备同步。

### 9.6 前端资源与依赖治理（低优先）

界面依赖多个第三方 CDN（Bootstrap、Font Awesome、MathJax、CodeMirror）。为提升离线可用性与加载稳定性，建议将关键依赖本地化托管并锁定版本，同时评估 MathJax 的按需加载以减少首屏体积。

### 9.7 学习数据价值挖掘（增量）

系统已记录题目查看行为并具备错题本，具备做个性化推荐与学情分析的数据基础。可在管理侧总览中增加错题趋势、薄弱知识点、按来源/年份的掌握度等统计，进一步发挥题库的备考价值。

## 10. 接手维护清单

恢复完整开发环境时，需从以下资产入手：

- 后端源码及路由处理逻辑
- `templates/` 全部页面模板
- `static/` 全部前端资源（JS、CSS、图标）
- `.tex` 试卷模板文件及 LaTeX 编译工具链配置
- 依赖清单（requirements.txt）
- 数据库 schema 或迁移文件，重点为题目、错题本、用户三者的关联关系
- 图片与 PDF 的存储路径及命名规则
- 部署与运行环境配置
